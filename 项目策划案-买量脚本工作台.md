# 买量脚本工作台 — 项目策划案

> **项目定位**：面向手游买量（UA）团队的 AI 驱动视频脚本生成工具  
> **核心理念**：数据驱动 + 钩子优先 — 前3秒决定一切  
> **当前形态**：单文件 HTML（约2355行），需拆分为模块化前端工程

---

## 一、项目概述

### 1.1 产品目标

为手游买量团队提供一站式视频脚本创作工作台，通过 AI 联网搜索热点趋势 + 智能匹配钩子套路 + 节奏蓝图自动编排，将分镜脚本生成效率提升数倍。

### 1.2 核心工作流

```
选平台 → 搜索热点（5维度） → 勾选主题 → 配置参数（游戏/时长/类型/画像/卖点）
→ AI 生成分镜脚本 → 导出（Seedance提示词 / Lovart分镜 / 文本）
```

### 1.3 目标用户

| 角色 | 使用场景 |
|------|----------|
| 买量创意策划 | 日常批量产出脚本，需要热点灵感 |
| 视频制作/剪辑 | 拿到分镜后执行落地 |
| 投放优化师 | 分析竞品素材、迭代优化方向 |
| 创意总监 | 审核脚本质量、把控创意方向 |

### M1 主题雷达（Theme Radar）

**位置**：左侧面板  
**职责**：通过 AI 联网搜索，从多个维度获取当前热点，为脚本创作提供灵感素材

#### M1.1 平台选择

| 功能 | 说明 |
|------|------|
| 平台切换 | 支持抖音、快手、B站、全平台 |
| 全部搜索 | 一键触发所有维度的搜索（间隔 800ms 防并发） |

#### M1.2 五维度搜索

每个维度对应独立的 AI 搜索提示词模板，返回 JSON 数组结构化数据：

| 维度 | 图标 | 搜索内容 | 返回字段 |
|------|------|----------|----------|
| 热门话题 | 🔥 | 当日热搜/热梗/社会事件 | name, desc, tag, heat, hooks[], cases[] |
| 爆款视频 | 🎬 | 近7天跑量最好的视频类型和套路 | 同上 |
| 热门角色 | ⭐ | 有流量的网红/IP/meme形象 | 同上 |
| 竞品动态 | 📈 | 同类游戏的投放策略和创意方向 | 同上 |
| 钩子套路 | 🎣 | 效果最好的前3秒钩子类型 | 同上 |

#### M1.3 预置主题库

14个预置引流主题，分3个梯队（T1/T2/T3）：

| 梯队 | 主题 |
|------|------|
| T1（高热） | 丧尸末日、塔防策略、拯救营救、倒计时挑战 |
| T2（中热） | 弹幕割草、巨型Boss战、地牢探险、生存逃脱、逆袭成长 |
| T3（常青） | 解压满足、狗血反转、怀旧经典、合成进化、PVP对决 |

#### M1.4 自定义主题

- 手动添加：输入名称 + 描述 + 类型分类
- 主题分类：题材美学、情绪共鸣、网络热梗、影视热点、爽感公式、反差猎奇

#### M1.5 主题卡片

每个主题卡片展示：
- 主题名称、描述
- 标签（类型标签 + 热度标签）
- 可展开详情：钩子做法 + 爆款案例
- 勾选/取消勾选（支持多选）

#### M1.6 搜索结果解析

- AI 返回 JSON 数组，自动解析为主题卡片
- 支持离线模式：复制提示词到外部 AI → 粘贴结果导入

### M4 钩子引擎（Hook Engine）

**职责**：管理14种经验证的前3秒钩子套路，根据选中主题智能匹配最佳钩子

#### M4.1 钩子套路库（14种）

| ID | 名称 | 核心机制 | 适用场景 |
|----|------|----------|----------|
| suspense | 悬念反问 | 抛出让人想知道答案的问题 | zombie, boss, dungeon, survive, pvp |
| number_shock | 数字冲击 | 用巨大数字变化制造视觉冲击 | bullet, ceo, merge, tower, countdown |
| instant_fail | 故意失败 | 角色第1秒就死亡，制造"不可能"感 | zombie, dungeon, survive, countdown, rescue |
| visual_overload | 视觉暴击 | 第一帧满屏爆炸/宝箱/金币 | bullet, tower, boss, merge, asmr |
| reversal_preview | 反转预告 | 先展示高潮，再倒回开头 | drama, ceo, survive, pvp, boss |
| contrast | 对比冲突 | 分屏或快切，用强烈反差制造冲突 | ceo, rescue, drama, pvp, merge |
| forbidden | 禁忌好奇 | 暗示"不该做"的事，利用好奇心 | dungeon, drama, rescue, survive, countdown |
| countdown_bomb | 倒计时紧迫 | 倒数计时制造紧迫感 | countdown, survive, boss, tower, pvp |
| asmr_crack | 解压碾碎 | 极致解压/碾碎/切割画面 | asmr, merge, bullet, tower |
| meme_hijack | 热梗劫持 | 用当下最火的梗开场 | drama, nostalgia, ceo, rescue |
| pov_fall | POV坠入 | 第一人称坠落/穿越 | dungeon, survive, zombie, boss |
| one_take | 一镜穿越 | 镜头不间断穿越多个场景 | dungeon, tower, bullet, merge, boss |
| scale_reveal | 尺度揭示 | 先小场景，突然拉远揭示全貌 | boss, tower, zombie, survive, bullet |
| wrong_game | 货不对板 | 开头像A类型，突然变成完全不同内容 | drama, nostalgia, rescue, ceo, merge |

每个钩子包含：id、名称、描述、适用场景ID列表、示例描述

#### M4.2 智能匹配算法

```
输入：用户选中的主题名称列表
处理：
  1. 将主题名映射到预置主题ID
  2. 遍历钩子库，计算每个钩子与选中主题的场景交集数量
  3. 按交集数量降序排序
  4. 返回Top 5（若高分不足3个则补全到5个）
输出：排序后的钩子列表，注入System Prompt
```

### M6 AI 对话引擎（Chat Engine）

**职责**：构建System Prompt、管理对话上下文、调用AI API、处理流式响应

#### M6.1 System Prompt 构建

System Prompt 由以下模块动态拼接：

```
├── 角色定义（顶级手游买量创意总监）
├── 第一原则：钩子决定一切
│   ├── 可用钩子套路（匹配后的Top5）
│   ├── 当前主题验证有效的钩子（来自搜索结果）
│   └── 爆款参考案例（来自搜索结果）
├── 第二原则：节奏蓝图
│   └── 按时长的段落分配表
├── 任务上下文
│   ├── 游戏信息（名称/类型/卖点）
│   ├── 卖点标签（已选中的卖点）
│   ├── 吸量主题（已勾选的主题）
│   ├── 目标用户画像
│   ├── 主角人设
│   ├── 联动IP（含搜索到的详细资料）
│   ├── 视频参数（时长/比例）
│   └── 构图指南（按比例）
├── 脚本类型（角色定义 + 输出格式 + 规则）
├── 输出格式要求
│   ├── 分镜脚本格式
│   └── Seedance 2.0 转换（按需）
└── 内容安全红线
```

#### M6.2 对话管理

| 功能 | 说明 |
|------|------|
| 消息列表 | AI消息（左对齐）+ 用户消息（右对齐，渐变背景） |
| 流式响应 | 支持 SSE 流式输出，实时更新消息气泡 |
| 降级机制 | 流式失败自动重试非流式请求 |
| 快捷按钮 | 生成分镜脚本 / 分析买量视频 / 换个方向 / 钩子推荐 / 推荐主角 |
| 会话归档 | 每次"生成脚本"自动归档当前对话 |
| 离线模式 | 未配置API时，复制提示词到剪贴板 → 粘贴AI回复导入 |

#### M6.3 API 调用流程

```
1. 获取API配置（task: 'search' | 'gen'）
2. 判断是否为手动模式（无API Key）
   → 是：复制提示词到剪贴板，提供粘贴导入入口
   → 否：继续
3. 构建请求
   - URL: OpenRouter / Perplexity / DeepSeek / OpenAI / 自定义
   - Headers: Content-Type + Authorization + (OpenRouter: HTTP-Referer, X-Title)
   - Body: model, messages, stream, temperature(0.85), max_tokens(4096)
4. 发送请求
   - 优先流式（stream: true）
   - 解析 SSE 数据流，实时回调 onChunk
5. 失败处理
   - 地区限制：提示切换模型
   - 流式失败：自动降级为非流式重试
6. 完成回调 onDone(text, error)
```

### M7 脚本生成与导出（Script Output）

**职责**：根据配置生成分镜脚本，提供多种导出格式

#### M7.1 脚本生成

生成时自动组装的 Prompt 包含：
- 所选主题
- 推荐钩子（Top3名称）
- 节奏蓝图段数
- 脚本类型要求和格式
- 口播类型的特殊强化提示（字数要求、全程不停嘴）
- 卖点标签
- 联动IP信息
- 爆款案例参考

#### M7.2 导出动作

| 动作 | 说明 |
|------|------|
| 📋 复制 | 复制完整脚本文本 |
| 💾 导出 | 下载为 .txt 文件 |
| 🇬🇧 转 Seedance 2.0 英文提示词 | AI将脚本压缩为≤2000字符英文提示词 |
| 🇨🇳 转 Seedance 2.0 中文提示词 | AI将脚本压缩为≤2000字符中文提示词 |
| 📋 导出 Lovart 分镜 | 解析分镜结构，格式化后复制（用于 Lovart.ai） |

#### M7.3 分镜解析器（Frame Parser）

支持4种格式的自动解析：
1. 时间轴格式：`0-3s【段落名】内容...`
2. 方括号格式：`【段落名】内容...`
3. 序号格式：`分镜1: 内容...` / `Shot 1: 内容...`
4. 数字列表格式：`1. 内容...`
5. 兜底：按空行分割段落

### M8 视频分析模块（Video Analysis）

**职责**：上传竞品/参考买量视频，AI深度结构化分析，指导脚本迭代

#### M8.1 视频上传

| 功能 | 说明 |
|------|------|
| 触发方式 | 点击"分析买量视频"按钮 |
| 文件限制 | 视频文件（video/*），最大 100MB |
| 预览 | 内嵌视频播放器 + 文件名/大小/时长/分辨率 |

#### M8.2 分析流程

```
1. 显示视频预览 + 分析方法标识
2. 点击"开始分析"
3. 帧提取（Canvas抽帧）
   - 帧率：2 FPS
   - 最大帧数：120帧
   - 分辨率：宽度≤960px，JPEG 75%质量
   - 进度条实时更新
4. 发送到阿里云千问 DashScope API
```

## 五、内容安全红线

脚本生成的所有内容必须遵守以下限制：

| 类别 | 规则 |
|------|------|
| 真人肖像 | 禁止使用真人照片/视频 |
| IP名称 | 禁止直接使用知名IP名称 |
| 武器 | 真实武器 → 替换为能量法器 |
| 血腥 | 禁止血腥画面 → 光点消散效果 |
| 恐怖 | 丧尸等恐怖元素需Q版化处理 |
| 赌博/色情 | 完全禁止 |
| 怪物死亡 | 统一用"化为光点消散+金币弹出" |
